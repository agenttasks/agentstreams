# Architecture Diagrams

Two diagram sets covering the same architecture, optimized for different readers.

## `human/` — Visual Diagrams (PNG)

Generated by `diagrams` ([mingrammer/diagrams](https://github.com/mingrammer/diagrams)) + Graphviz.

| File | What it shows |
|------|---------------|
| `system-overview.png` | Full system: CLI, webapp, orchestrator, 25 agents, data stores, vendors |
| `pipeline-flows.png` | All 8 pipelines with step ordering, parallelism, and gates |
| `14-layer-stack.png` | L0 Training → L10 Governance with vendor dependencies |
| `agent-topology.png` | 25 agents grouped by model tier (opus/sonnet/haiku) |

Regenerate: `uv run python diagrams/generate_human.py`

## `claude/` — Structured Diagrams (YAML)

Designed for LLM consumption. Dense, unambiguous, with explicit IDs and relationships.

| File | What it shows |
|------|---------------|
| `system-overview.yaml` | Entry points, orchestrator config, agent roster, data layer, vendors |
| `pipeline-flows.yaml` | All 8 pipelines with exact step ordering, conditions, gates, ASCII flows |
| `14-layer-stack.yaml` | 14 layers with module paths, exports, concepts, vendor/safety-research refs |
| `agent-topology.yaml` | 25 agents with tools, skills, connectors, safety-research refs, constraints |

### Usage with Claude

Point Claude at any YAML file for architecture reasoning:

```
Read diagrams/claude/pipeline-flows.yaml and explain which agents
run in parallel during the standard-codegen pipeline.
```

```
Read diagrams/claude/agent-topology.yaml and verify that no
read-only agent has Write or Edit in its tool grant.
```

## Dependencies

- Python: `diagrams` (in `[dependency-groups.diagrams]`)
- System: `graphviz` (`apt-get install graphviz`)
